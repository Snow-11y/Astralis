Instead of patching vanilla's aging OpenGL 2.1 renderer with incremental improvements, Astralis builds an entirely new graphics engine from the ground upâ€”one that speaks **Vulkan**, **DirectX 12**, **Metal**, and **modern OpenGL** fluently.

## ğŸ“¦ What Does Astralis Provide?

### ğŸ¨ Modern Graphics Engine
- **Multiple rendering backends**: Vulkan, OpenGL, DirectX 12, Metal - automatically selects the best for your hardware
- **Smart fallback system**: If one API fails, seamlessly switches to another without crashing
- **GPU-driven rendering**: Let your graphics card do the heavy lifting instead of your CPU
- **Advanced shader support**: SPIR-V, GLSL, HLSL, MSL - write shaders in any language

### âš¡ Performance Boost
- **Massive FPS improvements**: Eliminate CPU bottlenecks with intelligent batching and culling
- **Reduced memory usage**: Off-heap allocation means less garbage collection lag
- **Multi-threaded rendering**: Use all your CPU cores efficiently
- **Dynamic resolution scaling**: Automatically adjusts to maintain smooth framerates

### ğŸŒ Multi-Language Modding (PolyLang)
- **Write mods in 60+ languages**: Java, Kotlin, Python, JavaScript, Rust, and more
- **Cross-language compatibility**: Mods written in different languages work together seamlessly
- **DeepMix system**: Inject code into other mods regardless of what language they're written in
- **Simple API**: Clean, intuitive interfaces for mod developers

### ğŸ® Quality of Life Features
- **Better chunk loading**: Smoother world generation and loading
- **Enhanced entity rendering**: Smarter culling and LOD (Level of Detail) systems
- **Improved particle effects**: More particles with less performance impact
- **Modern Java features**: Built on Java 21+ with latest optimizations

### ğŸ›¡ï¸ Compatibility & Stability
- **Self-healing architecture**: Automatically recovers from errors and adapts to problems
- **Extensive mod compatibility**: Works alongside popular mods (untested but designed for it)
- **Detailed diagnostics**: Clear error messages and performance reports
- **Hot-reload support**: Change shaders and configs without restarting

### ğŸ¯ Why Astralis?

Astralis pushes Minecraft 1.12.2 beyond its limits by:

- **Using modern graphics technology** - Your GPU can do so much more than vanilla Minecraft lets it
- **Supporting multiple platforms** - Windows, Linux, macOS, even mobile (experimental)
- **Enabling multi-language modding** - Choose the programming language you're comfortable with
- **Staying compatible** - Designed to work with your favorite existing mods

**Want the technical details?** Check out the [Architecture Wiki](https://github.com/Snow-11y/astralis/wiki/Architecture).

## âš¡ Key Features

### ğŸ® Four-Engine Core Systems

Astralis consists of four major subsystems, each solving a specific problem:

| System | Purpose | Key Tech |
|:---|:---|:---|
| **â„ï¸ Snowflake** | Off-heap ECS for entity management | Panama FFI, SIMD acceleration |
| **ğŸŒ¨ï¸ Snowy** | Modern Minecraft features & QoL | Java 21+, Records, Pattern matching |
| **âš¡ Snowium** | Multi-backend rendering engine | Vulkan/GL/DX/Metal, GPU-driven |
| **ğŸŒªï¸ Snowstorm** | Extreme optimization mode | Batching, culling, async everything |

### ğŸ”¥ Performance Highlights

- **10,000+ draw calls â†’ 1 draw call** via batching and GPU culling
- **Zero GC pauses** on hot paths (off-heap memory)
- **Multi-threaded rendering** with structured concurrency
- **Dynamic resolution scaling** to maintain target FPS
- **Automatic backend fallback** if Vulkan fails

### ğŸŒ PolyLang & DeepMix

Write Minecraft mods in **60+ programming languages**:

```kotlin
// Kotlin mod
@Astralis("your_mod")
class yourMod : Astralis() {
    override fun onInitialize() {
        println("Hello from Kotlin!")
    }
}
```

```python
# Python mod
@astralis("your_python_mod")
class yourPythonMod:
    def on_initialize(self):
        print("Hello from Python!")
```

**DeepMix** allows you to inject code into any mod, written in any language, using Astralis's bytecode generation system.

For details, see the [PolyLang Wiki](https://github.com/Snow-11y/astralis/wiki/PolyLang).
